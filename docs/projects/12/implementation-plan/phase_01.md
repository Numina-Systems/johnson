# GH12: Dynamic System Prompt Provider - Phase 1

**Goal:** Extract system prompt building from the agent loop into a pluggable provider callback on `AgentDependencies`.

**Architecture:** Add an optional `systemPromptProvider` field to `AgentDependencies` that receives tool docs (generated fresh each call by the agent loop) and returns the assembled system prompt. The agent loop calls the provider when present, falls back to the existing inline logic when absent, and caches the last successful result for graceful degradation on provider failure.

**Tech Stack:** TypeScript (Bun runtime), strict mode

**Scope:** 2 phases from original design (this is phase 1 of 2)

**Codebase verified:** 2026-04-27

---

## Acceptance Criteria Coverage

This phase implements:

### GH12.AC1: Provider receives toolDocs and returns the complete system prompt
- **GH12.AC1.1 Success:** Provider function is called with the tool docs string generated by the registry and its return value is used as the system prompt for the model call

### GH12.AC2: Provider failure falls back to cached last-good prompt with stderr warning
- **GH12.AC2.1 Failure:** When the provider throws, the agent uses the cached last-good system prompt and writes a warning to stderr

### GH12.AC3: Existing behaviour preserved when no provider is set (inline fallback)
- **GH12.AC3.1 Success:** When `systemPromptProvider` is undefined, the agent builds the system prompt inline exactly as it does today

### Task 2: Refactor agent loop to use provider with fallback and caching

**Verifies:** GH12.AC1.1, GH12.AC2.1, GH12.AC3.1

**Files:**
- Modify: `src/agent/agent.ts` (lines 95-106, the prompt-building block inside `_chatImpl`)

**Implementation:**

This task replaces the inline prompt-building block in `_chatImpl` with a provider-first approach that falls back to inline logic.

**Step 1:** Add a `cachedSystemPrompt` closure variable inside `createAgent`, alongside the existing `history` and `currentContext` variables (around line 51-52):

```typescript
let cachedSystemPrompt = '';
```

This sits inside the `createAgent` closure so it persists across `chat()` calls for this agent instance.

**Step 2:** Replace the prompt-building block (lines 95-106 of `agent.ts`). The current code is:

```typescript
// a. Read persona
const persona = await Bun.file(deps.personaPath).text();

// b. Load core memory (self document) and list skill names
const coreMemory = loadCoreMemoryFromStore(deps.store);
const allDocs = deps.store.docList(500);
const skillNames = allDocs.documents
  .filter(d => d.rkey.startsWith('skill:'))
  .map(d => d.rkey);

// c. Build system prompt (now includes tool docs)
const systemPrompt = buildSystemPrompt(persona, coreMemory, skillNames, toolDocs, deps.config.timezone);
```

Replace it with:

```typescript
// Build system prompt via provider (if set) or inline fallback
let systemPrompt: string;
if (deps.systemPromptProvider) {
  try {
    systemPrompt = await deps.systemPromptProvider(toolDocs);
    cachedSystemPrompt = systemPrompt;
  } catch (err) {
    process.stderr.write(`[agent] system prompt provider failed, using cached: ${err instanceof Error ? err.message : err}\n`);
    systemPrompt = cachedSystemPrompt;
  }
} else {
  // Inline fallback â€” preserves behaviour when no provider is set
  const persona = await Bun.file(deps.personaPath).text();
  const coreMemory = loadCoreMemoryFromStore(deps.store);
  const allDocs = deps.store.docList(500);
  const skillNames = allDocs.documents
    .filter(d => d.rkey.startsWith('skill:'))
    .map(d => d.rkey);
  systemPrompt = buildSystemPrompt(persona, coreMemory, skillNames, toolDocs, deps.config.timezone);
}
```

**Key design decisions:**
- The `else` branch is identical to the current code, so agents created without a `systemPromptProvider` behave exactly as before (GH12.AC3.1).
- On first call with a provider, if the provider throws before ever succeeding, `cachedSystemPrompt` is `''`. This is acceptable -- the model will get an empty system prompt, which is recoverable (the user will see the error on stderr and can fix the provider). This avoids the complexity of eagerly building a fallback prompt that might never be needed.
- The cache is per-agent-instance (closure variable), not global. Each agent gets its own cache.
